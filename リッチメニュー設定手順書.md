# タブ切り替え式リッチメニュー設定 手順書

対象：「プレ交際」「真剣交際」の2タブ切り替えリッチメニュー
作業場所：パソコンのターミナル（Mac）またはコマンドプロンプト／PowerShell（Windows）

---

## 0. 事前に準備するもの

| 準備するもの | 入手方法 |
|---|---|
| チャネルアクセストークン | [LINE Developers Console](https://developers.line.biz/console/) → 対象チャネル → 「Messaging API設定」タブ → 下の方にある「チャネルアクセストークン（長期）」の「発行」ボタン |
| リッチメニュー画像2枚 | IMG_2902.png（プレ交際タブ）、IMG_2903.png（真剣交際タブ）※すでにお持ちのもの |
| 各カードのLINEミニアプリURL（LIFF ID） | LINE Developers Console → 該当のLINEミニアプリのチャネル → 「LIFF」タブ → 各LIFFアプリの「LIFF ID」をコピー |
| curlコマンドが打てる環境 | Mac：標準の「ターミナル」アプリでOK<br>Windows：「コマンドプロンプト」または「PowerShell」（curlは標準搭載） |

画像2枚は、作業する際にわかりやすい場所（例：デスクトップに`richmenu`という名前のフォルダを作る）に置いておいてください。

---

## 1. 作業フォルダに移動する

ターミナル（コマンドプロンプト）を開き、画像を置いたフォルダに移動します。

```bash
cd Desktop/richmenu
```

（フォルダ名・場所はご自身の環境に合わせて変更してください）

このフォルダの中に `richmenu-pre.png`（IMG_2902.pngをリネーム）と `richmenu-shinken.png`（IMG_2903.pngをリネーム）を置いておくと、この後のコマンドがそのままコピペで使えます。

---

## 2. リッチメニューAを作成する【プレ交際】

以下のコマンドをテキストエディタ（メモ帳やVSCodeなど）に貼り付け、`{channel access token}` の部分だけをご自身のトークンに書き換えてから、**まるごとターミナルに貼り付けてEnter**します。

```bash
curl -v -X POST https://api.line.me/v2/bot/richmenu \
-H 'Authorization: Bearer {channel access token}' \
-H 'Content-Type: application/json' \
-d '{
    "size": { "width": 2500, "height": 1686 },
    "selected": true,
    "name": "richmenu-pre",
    "chatBarText": "メニュー",
    "areas": [
        {
            "bounds": { "x": 1250, "y": 0, "width": 1250, "height": 223 },
            "action": {
                "type": "richmenuswitch",
                "richMenuAliasId": "richmenu-alias-shinken",
                "data": "switch-to-shinken"
            }
        },
        {
            "bounds": { "x": 0, "y": 223, "width": 836, "height": 998 },
            "action": { "type": "uri", "uri": "https://miniapp.line.me/{自己開示Part1のliffId}" }
        },
        {
            "bounds": { "x": 836, "y": 223, "width": 827, "height": 998 },
            "action": { "type": "uri", "uri": "https://miniapp.line.me/{自己開示Part2のliffId}" }
        },
        {
            "bounds": { "x": 1663, "y": 223, "width": 837, "height": 998 },
            "action": { "type": "uri", "uri": "https://miniapp.line.me/{自己開示Part3のliffId}" }
        },
        {
            "bounds": { "x": 0, "y": 1221, "width": 1250, "height": 465 },
            "action": { "type": "uri", "uri": "https://miniapp.line.me/{婚活プロフィールのliffId}" }
        },
        {
            "bounds": { "x": 1250, "y": 1221, "width": 1250, "height": 465 },
            "action": { "type": "uri", "uri": "https://miniapp.line.me/{価値観すり合わせのliffId}" }
        }
    ]
}'
```

**実行すると、画面に以下のような結果が返ってきます。**

```json
{"richMenuId":"richmenu-xxxxxxxxxxxxxxxxxxxx"}
```

👉 この `richmenu-xxxxxxxxxxxxxxxxxxxx` の部分を**必ずメモ帳などにコピーして保存**しておいてください。これを以後 `{richMenuId_A}` として使います。

---

## 3. リッチメニューAに画像をアップロードする

先ほどメモした `{richMenuId_A}` に書き換えてから実行します。

```bash
curl -v -X POST https://api-data.line.me/v2/bot/richmenu/{richMenuId_A}/content \
-H 'Authorization: Bearer {channel access token}' \
-H "Content-Type: image/png" \
-T richmenu-pre.png
```

- `{richMenuId_A}` → 手順2でメモしたID
- `{channel access token}` → 手順0のトークン
- `-T richmenu-pre.png` の部分は、今いるフォルダに画像があればこのままでOK。フォルダが違う場合はファイルまでのパスを指定してください。

エラーが出ず何も表示されなければ成功です。

---

## 4. リッチメニューBを作成する【真剣交際】

同様に、テキストエディタで `{channel access token}` を書き換えてから実行します。

```bash
curl -v -X POST https://api.line.me/v2/bot/richmenu \
-H 'Authorization: Bearer {channel access token}' \
-H 'Content-Type: application/json' \
-d '{
    "size": { "width": 2500, "height": 1686 },
    "selected": false,
    "name": "richmenu-shinken",
    "chatBarText": "メニュー",
    "areas": [
        {
            "bounds": { "x": 0, "y": 0, "width": 1250, "height": 223 },
            "action": {
                "type": "richmenuswitch",
                "richMenuAliasId": "richmenu-alias-pre",
                "data": "switch-to-pre"
            }
        },
        {
            "bounds": { "x": 0, "y": 223, "width": 836, "height": 1463 },
            "action": { "type": "uri", "uri": "https://miniapp.line.me/{真剣交際後の確認のliffId}" }
        },
        {
            "bounds": { "x": 836, "y": 223, "width": 827, "height": 1463 },
            "action": { "type": "uri", "uri": "https://miniapp.line.me/{家条件すり合わせのliffId}" }
        },
        {
            "bounds": { "x": 1663, "y": 223, "width": 837, "height": 1463 },
            "action": { "type": "uri", "uri": "https://miniapp.line.me/{家族紹介のliffId}" }
        }
    ]
}'
```

ここでも `{"richMenuId":"richmenu-yyyyyyyyyyyyyyyyyyyy"}` が返ってくるので、これを `{richMenuId_B}` としてメモしておきます。

---

## 5. リッチメニューBに画像をアップロードする

```bash
curl -v -X POST https://api-data.line.me/v2/bot/richmenu/{richMenuId_B}/content \
-H 'Authorization: Bearer {channel access token}' \
-H "Content-Type: image/png" \
-T richmenu-shinken.png
```

- `{richMenuId_B}` → 手順4でメモしたID

---

## 6. デフォルトメニューを設定する（最初にAを表示させる）

友だち追加時などに最初に表示させたいメニュー（今回はA＝プレ交際）を指定します。

```bash
curl -v -X POST https://api.line.me/v2/bot/user/all/richmenu/{richMenuId_A} \
-H 'Authorization: Bearer {channel access token}'
```

---

## 7. エイリアスを作成する（タブ切り替えの本体）

これが2つのメニューを紐づけて「切り替え」を可能にする設定です。**2つとも**実行してください。

```bash
curl -v -X POST https://api.line.me/v2/bot/richmenu/alias \
-H 'Authorization: Bearer {channel access token}' \
-H 'Content-Type: application/json' \
-d '{
    "richMenuAliasId": "richmenu-alias-pre",
    "richMenuId": "{richMenuId_A}"
}'
```

```bash
curl -v -X POST https://api.line.me/v2/bot/richmenu/alias \
-H 'Authorization: Bearer {channel access token}' \
-H 'Content-Type: application/json' \
-d '{
    "richMenuAliasId": "richmenu-alias-shinken",
    "richMenuId": "{richMenuId_B}"
}'
```

---

## 8. 動作確認

1. 自分のLINE公式アカウントを友だち追加し直す、またはトーク画面を開き直す
2. リッチメニューが画像1（プレ交際タブ）の状態で表示される
3. 右上「真剣交際」をタップ → 画像2（真剣交際タブ）に切り替わる
4. 各カード（自己開示Part1〜3、婚活プロフィールなど）をタップ → 該当のLINEミニアプリが開く

これがすべて動けば設定完了です。

---

## つまずきやすいポイント

| 症状 | 原因の可能性 |
|---|---|
| `{"message":"Authentication failed..."}` と出る | チャネルアクセストークンの入力ミス、コピー漏れ |
| 画像アップロードで `Invalid image format` | 画像がPNG/JPEGでない、または1MBを超えている |
| メニューは出るがタップしても反応しない | LIFF IDの書き換え忘れ（`{◯◯のliffId}`のまま送信している） |
| タブが切り替わらない | 手順7のエイリアス作成を忘れている、または`richMenuAliasId`の綴りが2つのメニューで一致していない |
| 全部設定したのに古いメニューのまま | LINEアプリ側のキャッシュ。トークルームを一度閉じて再度開く、または友だちのLINEアプリを再起動 |

---

## 実行順序まとめ

1. トークン・LIFF ID・画像を準備
2. リッチメニューA作成 → `richMenuId_A`を記録
3. Aに画像アップロード
4. リッチメニューB作成 → `richMenuId_B`を記録
5. Bに画像アップロード
6. デフォルトメニュー設定（A）
7. エイリアス作成（pre／shinken 両方）
8. 実機で動作確認
